# /audit - Documentation Audit

Use this workflow when existing documentation (previous specs, notes, inherited docs) is present.

## Steps

1. **Inventory**: List all relevant documentation files.
2. **Apply Clarity Test**: For each section, ask:
    - **Actionable?** (Can AI act on this? If not, delete.)
    - **Current?** (Is this still the decision?)
    - **Single Source?** (Is this said elsewhere? Consolidate.)
3. **Prune**:
    - Remove "vision" and "future state" (e.g., "In the future we might...")
    - Delete motivational preambles.
    - Remove speculative features.
4. **Target**: Aim for a 40-50% reduction in volume while keeping all actionable data.

> "A lean spec is a fast spec."
